Because of a long outstanding bug/feature request in View Engine, it has is not been possible to lazy load entry components. This limitation could be worked around by using Web Components and Angular Elements. With the introduction of the Ivy Rendering Engine this is no longer needed (as Entry Components have been depreciated)

Define a 'registry' of modules to be lazy loaded. The import path follows the same convention as when defining `loadChildren` Just like when defining lazy loaded child routes. The Angular Compiler will take care of code splitting for you. Then provide the registry as a provider for the parent module. 

{{ moduleProviders }}

Create a ModuleLoadingService to load the module when required.

NOTE: The code example of the module loading service below is simplified to emphasize the key points. Please see TODO Module Loading Service for full details 

{{ moduleLoadingService }}

The lazy loading strategy (scrolling, dialog etc) will determine what to do once the module is loaded and compiled. For this example, The module loading service produces an Observable of the component factory so the scrolling component can subscribe and render the entry component once loaded.

NOTE: A LazyLoadScroll Directive is used to create the component from the factory. It makes use of the ResizeObserver API to listen element size changes. ResizeObserver is not currently widely supported and TypesScript does not have a definition for it, hence A polyfill is currently being used. See Can I Use and TypeScript

Try it Out

  1. Open Developer Tools -> Networking Tab
  2. Clear History
  3. Refresh Browser Window
  4. Slowly Start Scrolling
  5. Note the lazy loading of modules as you scroll
  6. Profit!!
