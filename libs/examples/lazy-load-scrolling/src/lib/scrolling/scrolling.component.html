Because of a long outstanding bug/feature request in Angulars View Engine, it has is not been possible to lazy load entry components. This limitation could be worked around by using Web Components and Angular Elements. With the introduction of the Ivy Rendering Engine this is no longer needed (as entry components have been depreciated)

Define a 'registry' of modules to be lazy loaded. The import path follows the same convention as when defining a route based lazy loaded module via the loadChildren route property. The Angular Compiler & Webpack will take care of code splitting for you. Then provide the registry as a provider to the parent module. 

{{ moduleProviders }}

Create a ModuleLoadingService to handle loading the modules as required.

NOTE: The code example of the module loading service below is simplified to emphasize the key points. Please see TODO Module Loading Service for full details 

{{ moduleLoadingService }}

The lazy loading strategy (scrolling, dialog etc) will determine what to do once the module is loaded and compiled. For this example, The module loading service produces an Observable of the component factory so the scrolling component can subscribe and render the entry component once loaded.

Try it Out

  1. Open Developer Tools -> Networking Tab
  2. Clear Network History
  3. Refresh Browser Window
  4. Start Scrolling
  5. Note the lazy loading of modules as you scroll
  6. Profit!!
